FAQ

UltraVNC (direkomendasikan). Kompatibel dengan TightVNC & viewer lain melalui argumen CLI yang bisa dikustom.
Hak admin diperlukan untuk kontrol layanan/restart. Untuk membuka viewer VNC biasa tidak selalu diperlukan.
Mendukung auto‑update diam (GitHub/FTP) dengan verifikasi file. Bisa dinonaktifkan untuk lingkungan offline.

Dokumentasi

Argumen CLI

vnc-launcher.exe --host 10.1.2.3 --port 5900 --password ***** --scale=90 --readonly
# atau jalankan profil
vnc-launcher.exe --profile "Kantor-Pusat"

Auto‑Update

Dukungan sumber GitHub/FTP. Validasi checksum & rollback jika gagal.

{
  "update": {
    "source": "github",
    "repo": "org/vnc-launcher",
    "channel": "stable"
  }
}

Kontrol Layanan

# Windows Service
sc stop uvnc_service
sc start uvnc_service
shutdown /r /t 0  # restart mesin

Keamanan

  • Role-based access (Admin/Operator/Viewer)
  • Penyimpanan password terenkripsi lokal
  • Opsi PSK atau sertifikat untuk VPN
